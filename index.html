Pier

Pier is a new programming language that compiles to a custom VM platform.
It is statically but weakly typed, procedural, and very similar to C.

About the Project

The Pier project is an attempt to create guaranteed platform-independence by creating a platform that supports 3D graphics, audio, networking, and many other things natively. It doesn't allow FFI and, with a strong focus on GPU programming, attempts to be compatible with any graphics API an implementation might use as its backend. This project has been in development for 4 years.

Availability

An outdated version of the standard library is available on GitHub at github.com/gabecampb/pier. The compiler and VM are not yet released.

Platform Features

• Graphics API (compatible with both modern and old graphics APIs)
• Shaders, represented in a bytecode format
• GPU computing
• Multithreading
• File I/O
• SIMD
• Multi-display support
• Controller, touchscreen, keyboard & mouse support
• Device orientation and position support
• Image capture from display or cameras
• 2D/3D audio
• Sound transmission, absorption, and scattering
• Audio file creation and editing
• Audio recording
• Audio playback
• Video file creation and editing
• Image file creation and editing
• Video playback
• Image loading
• Ray tracing
• Networking (TCP + UDP)

Roadmap

Access to all of the platform's features will be exposed through libraries over time.
Short circuit evaluation will be added and the compiler will be optimized for performance & cleaner code generation.
Compiler error messages will also be improved.

The VM's shader bytecode to GLSL translator is still a bit of a work in progress. Hopefully the VM will receive JIT support in the future for a significant performance improvement, but completion of a few subsystems of the VM (such as shader translation, CPU compute shader emulation) is taking priority.
